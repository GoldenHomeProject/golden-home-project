## Failure 1: Watchdog flags Pi-side network blips as service outages

**Evidence:**
- `{"t":"2026-05-17T07:12:27-04:00","label":"goldenhomeproject.com",...,"http":"000000","status":"fail"}` + identical entries for `meta_graph_api` and `ntfy_server` at the **same second** — three independent external providers cannot all fail simultaneously; this is local connectivity. Repeats at 07:13:29 and 07:14:29.
- The `http` field is `"000000"` (six zeros), not `"000"`. That string can only be produced when curl exits non-zero AND `|| echo "000"` also fires — i.e. it's curl's `%{http_code}=000` (connection-level failure) concatenated with the fallback, never a real HTTP response. (See `pi/watchdog/ghp-watchdog.sh:58`.)
- 15+ such entries across `ntfy_server`, `goldenhomeproject.com`, `meta_graph_api` in 7 days — all isolated 60-second blips on three unrelated providers. Pattern is the local network, not the targets.

**Root cause:** `check_one` runs curl once with no retry; a single dropped packet on the Pi's link produces a recorded `fail`, and at the 07:12 cluster it crossed the 2-fail threshold and would have paged.

**Patch:** `pi/watchdog/ghp-watchdog.sh:55-64`

```diff
--- a/pi/watchdog/ghp-watchdog.sh
+++ b/pi/watchdog/ghp-watchdog.sh
@@ -55,7 +55,10 @@ notify() {
 check_one() {
   local label="$1" url="$2" regex="$3"
   local code
-  code=$(curl -sS -o /dev/null -w '%{http_code}' --max-time 10 "$url" 2>/dev/null || echo "000")
+  code=$(curl -sS -o /dev/null -w '%{http_code}' \
+    --max-time 10 --retry 2 --retry-delay 2 --retry-connrefused \
+    "$url" 2>/dev/null) || code="000"
   if [[ "$code" =~ $regex ]]; then
     echo "ok|$code"
   else
```

**Why safe:** `--retry` only re-attempts on transient/connection errors and 5xx — it does not retry 4xx, so the existing "Meta Graph 400 = reachable" semantics are preserved; worst-case wall time is ~30s, still inside the 60s loop interval; the `|| code="000"` rewrite normalises the doubled-zero log artifact without changing the regex match (still `000`, still `fail`); no new tools, no new tokens, no infra change.

**Verification:** After deploy (`sudo systemctl restart ghp-watchdog`), tail `~/ghp-watchdog/logs/checks.jsonl` for 24h — singleton `"http":"000"` entries should drop to ~zero. Force a real outage by `sudo iptables -A OUTPUT -d graph.facebook.com -j DROP` for 90s and confirm two consecutive fails still cross the threshold and fire the ntfy push.
